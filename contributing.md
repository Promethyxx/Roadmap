Contributing
First off, thank you for considering contributing to Oxy!

🌍 Translations
The best way to contribute right now is to help us with translations.
If you are fluent in multiple languages, your help is invaluable to make this project accessible to everyone.

💡 Share Your Ideas
We value your feedback and suggestions.
Please share your ideas or feature requests by opening an issue on the concerned project.

🐛 Reporting Bugs
If you’ve encountered a bug, please help us fix it by providing as much detail as possible.
When submitting a report, make sure to include:

Version: The version of the software you are using.

OS: Your operating system (e.g., Windows 11, macOS Sonoma, Ubuntu 24.04).

Logs: Please attach or paste your application logs.

🛠 Pull Requests & Self-Testing
If you are submitting code or a fix, you are responsible for testing it before opening a PR.

Use your own GitHub Actions or local environment to auto-test and build your branch.

Proof of Work: Please provide a screenshot or a log snippet showing that your build and tests passed successfully.

Quick checklist for Pull Requests:
Fork the repository and create your branch from main.

Ensure your code follows the existing style of the project.

Write a clear, concise commit message.

Attach proof that the code runs correctly in your environment.
